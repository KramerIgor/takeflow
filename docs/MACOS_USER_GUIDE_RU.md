# Takeflow для macOS

[English](MACOS_USER_GUIDE.md) | **Русский**

## Установка

1. Откройте [последний релиз Takeflow на GitHub](https://github.com/KramerIgor/takeflow/releases/latest).
2. Для Mac с M1/M2/M3/M4 или более новым чипом Apple скачайте `AppleSilicon.dmg`.
3. Для Mac с процессором Intel скачайте `Intel.dmg`.
4. Откройте загруженный DMG.
5. Перетащите **Takeflow** в **Applications / Программы**.
6. Откройте Takeflow из папки Applications.

Python, Homebrew, Node.js, Git и терминал не требуются.

## Первый запуск неподписанной beta-версии

Takeflow подписан ad-hoc для проверки целостности bundle, но не зарегистрирован и не нотарифицирован Apple. Поэтому macOS обычно блокирует первый запуск.

1. Попробуйте открыть Takeflow один раз и закройте предупреждение.
2. Откройте **Системные настройки → Конфиденциальность и безопасность**.
3. Прокрутите до раздела **Безопасность**.
4. Нажмите **Всё равно открыть / Open Anyway** рядом с Takeflow.
5. Подтвердите запуск и при необходимости введите пароль пользователя Mac.

После этого macOS запомнит исключение именно для Takeflow. Не отключайте Gatekeeper глобально и не запускайте скачанные команды для обхода защиты. Сравните SHA-256 DMG с файлом `.sha256`, приложенным к тому же релизу.

Официальное описание Apple: [открытие приложения неизвестного разработчика](https://support.apple.com/guide/mac-help/mh40616/mac).

## Локальные данные

- Настройки и история: `~/Library/Application Support/Takeflow`
- Лог запуска: `~/Library/Logs/Takeflow/launcher.log`
- Проекты по умолчанию: `~/Movies/Takeflow`

На вкладке **Проекты** можно выбрать другую доступную для записи корневую папку. Takeflow не переносит существующие проекты автоматически.

## Обновления

Takeflow автоматически выбирает пакет для архитектуры текущего Mac. После скачивания DMG приложение откроет его в Finder. Закройте Takeflow и замените старое приложение в Applications новой версией.

## Удаление

1. Закройте Takeflow кнопкой **Выход**.
2. Переместите `Takeflow.app` из Applications в Корзину.
3. Пользовательские проекты не удаляются автоматически.

При необходимости настройки и историю можно удалить отдельно из `~/Library/Application Support/Takeflow`, но только после резервного копирования нужных данных.
