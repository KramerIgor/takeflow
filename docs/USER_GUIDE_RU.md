# Руководство пользователя Takeflow

[English](USER_GUIDE.md) | **Русский**

Это руководство предназначено для обычной установки Takeflow на Windows. Python, Node.js и Git пользователю не нужны.

## Системные требования

- Windows 10 или Windows 11, 64-bit
- современный браузер
- подключение к интернету
- API-ключ Segmind и достаточный баланс для платных генераций
- не менее 250 МБ свободного места для приложения
- дополнительное место для видео и файлов проектов
- свободный локальный порт 7860

Установщик занимает около 20 МБ, установленное приложение — около 45 МБ без пользовательских данных.

## Установка

1. Откройте [последний релиз Takeflow](https://github.com/KramerIgor/takeflow/releases/latest).
2. Скачайте TakeflowSetup-*.exe.
3. Запустите установщик.
4. Выберите папку установки.
5. При необходимости оставьте включённым создание ярлыка.
6. Завершите установку и запустите Takeflow.

Beta-установщик пока не подписан цифровым сертификатом. Windows SmartScreen может показать предупреждение о неизвестном издателе. Проверяйте, что файл скачан из официального репозитория KramerIgor/takeflow, и сравнивайте SHA-256 с релизом.

## Запуск и остановка

Takeflow запускается с рабочего стола, из меню «Пуск» или через Takeflow.exe. Приложение поднимает локальный сервер и открывает:

http://127.0.0.1:7860

Для корректной остановки используйте кнопку **Выход** в верхней панели. Закрытие вкладки браузера не всегда останавливает приложение.

## Первая настройка

На вкладке **Проекты**:

1. Вставьте API-ключ Segmind.
2. Проверьте API base URL.
3. Выберите модель по умолчанию.
4. Выберите корневую папку для результатов.
5. Создайте или активируйте проект.

API-ключ хранится локально и не показывается после сохранения. Не публикуйте файл .env и скриншоты с ключом.

## Одиночная генерация

1. Откройте **Одиночная генерация**.
2. При необходимости задайте имя.
3. Введите промпт.
4. Добавьте референсы кнопкой или перетаскиванием.
5. Выберите модель, длительность, разрешение и соотношение сторон.
6. Проверьте предварительную стоимость.
7. Нажмите кнопку платной генерации.
8. Прочитайте и подтвердите предупреждение.

Задача сразу появляется в истории справа. Обновление истории не очищает текущий промпт.

## Референсы

- Для текущих Seedance 2.0 Base, Mini и Fast действует ограничение 9 файлов.
- Ограничение зависит от модели и может измениться для новых моделей.
- Изображения передаются через поддерживаемый API-процесс.
- Видео и аудио могут сохраняться с задачей, даже если текущий API-путь их не отправляет.
- Введите @ в промпте, чтобы вставить токен прикреплённого файла.
- Для удаления наведите курсор на превью и нажмите крестик.

Не перемещайте референсы, пока они нужны задаче в очереди.

## Очередь

В очередь можно заранее добавить несколько задач, проверить параметры и только потом запустить обработку. Создание задач очереди само по себе не является платной генерацией.

CSV import поддерживает безопасный предварительный просмотр. Сначала проверьте CSV, затем подтвердите создание задач. Запуск платной очереди выполняется отдельным действием.

## История

В карточках истории доступны статус, промпт, референсы, параметры, стоимость, результат, папка файлов, редактирование и перегенерация.

Перегенерация создаёт новый платный запрос после подтверждения и не перезаписывает старый результат.

## Стоимость

Предварительная стоимость рассчитывается локально по известным публичным тарифам. Итоговое списание провайдера может отличаться. Если надёжной цены нет, Takeflow показывает отсутствие оценки, а не выдуманное значение.

Перед отправкой всегда проверяйте предупреждение о платном действии и баланс Segmind.

## Обновления

Takeflow получает сведения о версиях из публичного update.json. Приложение не использует git pull.

При наличии обновления:

1. Запустите скачивание в панели обновления.
2. Дождитесь 100%.
3. Подтвердите запуск установщика.
4. Завершите установку.

Установщик не запускается без подтверждения пользователя.

## Решение проблем

Если браузер не открылся, вручную откройте http://127.0.0.1:7860.

Если порт занят, закройте другой экземпляр Takeflow. При необходимости завершите только старый Takeflow.exe через Диспетчер задач.

Лог запуска:

~~~text
%LOCALAPPDATA%\Takeflow\logs\launcher.log
~~~

Перед публикацией лога проверьте, нет ли в нём приватных локальных путей.

При ошибке генерации проверьте API-ключ, баланс, интернет, существование референсов и текст ошибки в деталях задачи. Не подтверждайте повторные платные запросы, пока статус предыдущего запроса неясен.

## Удаление

Откройте **Параметры Windows → Приложения → Установленные приложения → Takeflow → Удалить**.

Проекты сохраняются в выбранной пользователем корневой папке. Не используйте удаление приложения как способ очистки проектов.
